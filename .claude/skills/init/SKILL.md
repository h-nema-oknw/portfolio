# プロジェクト初期化

## 概要

このスキルは新しいプロジェクトで本フレームワークを使い始めるための初期化コマンドです。
`.claude/` と `CLAUDE.md` を配置済みの状態で実行してください。

実行後に作成されるもの：
```
agents/
    criteria.md        ← 評価指標（固定ファイル）
    phases/            ← 各フェーズの plan.md / task.md 格納先
docs/
    refs/              ← 参考資料置き場
    spec/              ← 仕様書格納先（Phase 1 で生成）
result/                ← 成果物格納先（Phase 2 以降で生成）
reports/               ← フェーズレポート格納先
```

---

## 実行手順

### Step 1 — 既存状態の確認

以下を確認し、ユーザーに状況を伝える：
- `CLAUDE.md` が存在するか（フレームワーク導入済みの確認）
- 既に `agents/` や `docs/` が存在する場合は上書きしない旨を確認する

`CLAUDE.md` が存在しない場合は以下を表示して中断する：
```
CLAUDE.md が見つかりません。
.claude/ フォルダと CLAUDE.md をプロジェクトルートに配置してから再実行してください。
```

---

### Step 2 — ディレクトリ作成

以下のディレクトリを作成する（既存の場合はスキップ）：

- `agents/phases/`
- `docs/refs/`
- `docs/spec/`
- `result/`
- `reports/`

---

### Step 3 — agents/criteria.md の配置

`agents/criteria.md` が存在しない場合のみ、以下の内容で作成する：

```markdown
# 評価指標

評価は **2段階** で行う。Stage 1・Stage 2 それぞれ **100点満点中80点以上** を合格とする。
両方の Stage が合格して初めて、そのフェーズを通過とみなす。

---

## Stage 1 — プラン評価（100点満点・80点以上で通過）

**目的**: エージェントが生成したプランが、タスク指示・仕様・ルールに適合しているかを評価する。

### 1-1. タスク網羅性（30点）

`docs/phases.md` に記載された対象フェーズの全タスクがプランに含まれているか。
- 全タスクが漏れなく含まれている（30点）
- 一部のタスクが欠けている（15点）
- 大半のタスクが欠けている / プランが不完全（0点）

### 1-2. 制約・ルール準拠（25点）

`CLAUDE.md` に定められた制約（禁止事項を含む）を守るプランになっているか。
- 全ての制約を守っている（25点）
- 一部の制約が考慮されていない（12点）
- 制約に反する内容を含む（0点）

### 1-3. 仕様整合性（25点）

`docs/spec/basic.md`・`docs/spec/detail.md` の要件とプランが矛盾していないか。
- 仕様と完全に整合している（25点）
- 一部の仕様と整合していない（12点）
- 仕様と大きく矛盾している（0点）

### 1-4. 実行可能性・具体性（20点）

プランが具体的で、エージェントが迷わず実行できるレベルになっているか。
- 全ての手順が明確で実行可能（20点）
- 一部の手順が曖昧または不明確（10点）
- 抽象的すぎて実行の指針にならない（0点）

---

## Stage 2 — 成果物評価（100点満点・80点以上で合格）

**目的**: 生成された成果物が、Stage 1 を通過したプランの内容に適合しているかを評価する。

### 2-1. プラン実装網羅性（40点）

プランに記載された全タスクが成果物に反映されているか。
- 全タスクが漏れなく実装されている（40点）
- 一部のタスクが実装されていない（20点）
- 大半のタスクが未実装（0点）

### 2-2. 品質・正確性（30点）

実装の完成度・正確性（`docs/spec/detail.md` の仕様値の正確な使用、エラーなし等）。
- 仕様値が正確に使われており、エラーなく動作する（30点）
- 軽微な誤りや不完全な箇所がある（15点）
- 重大なエラーや仕様値の誤りがある（0点）

### 2-3. 制約遵守（30点）

`CLAUDE.md` の制約が成果物においても守られているか。
- 全ての制約を守っている（30点）
- 一部の制約が守られていない（15点）
- 制約に反する実装を含む（0点）

---

## 合格条件

- **Stage 1** スコア ≥ 80点　かつ
- **Stage 2** スコア ≥ 80点

## 改善ループの動作

- Stage 1 不合格 → プランを修正して Stage 1 を再評価する
- Stage 2 不合格 → 成果物を修正して Stage 2 を再評価する（プランは維持）
- 各 Stage につき最大 **5回** の改善試行を行う
```

---

### Step 4 — 完了メッセージ

```
## プロジェクト初期化完了

作成したディレクトリ・ファイル：
- agents/criteria.md
- agents/phases/
- docs/refs/
- docs/spec/
- result/
- reports/

次のステップ：
1. 参考資料があれば docs/refs/ に配置してください。
2. /phase1-interview でプロジェクトの要件定義を開始してください。
```
